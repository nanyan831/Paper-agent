# Paper Agent MVP 真实试用 Checklist

## 1. 本轮试用目标

本 checklist 用来判断 Paper Agent 是否已经可以拿给真实用户做第一轮 MVP 试用。第一轮只验证一个闭环：

```text
导入 PDF
  -> 围绕真实任务提问
  -> AI 返回带出处的回答
  -> 来源显示页码和原文片段
  -> 用户点击回到 PDF 原文页
```

本轮不验证复杂文献图谱、多人协作、自动写完整论文、精美笔记系统。它们即使做出来，也不能证明当前最强痛点是否成立。

核心问题：

> 用户是否因为“每个 AI 回答都能回到原文页码”而愿意改变当前 ChatGPT/Kimi/豆包单篇上传加浏览器搜索的工作流？

## 2. 试用前 P0 功能验收清单

只有下表全部通过，才建议找真实用户正式试用。否则先做内部预试用。

| 模块 | P0 验收项 | 通过标准 | 失败表现 | 状态 |
|---|---|---|---|---|
| PDF 导入 | 用户能上传真实 PDF | 至少 10 篇 PDF 可连续导入，不需要开发者介入 | 只能单篇艰难上传，或用户不知道入口 | 待验收 |
| 导入反馈 | 显示解析结果 | 每篇资料显示标题、页数或 chunk 数、解析状态 | 上传后没有结果，用户问“去哪了” | 待验收 |
| 最近导入入口 | 用户能找到刚导入资料 | 首页或资料页能看到最近导入 PDF，并能打开 | 只能靠搜索找刚上传的论文 | 待验收 |
| PDF 阅读器 | 能打开原文 | 点击资料后可打开 PDF，翻页、缩放基本可用 | PDF 空白、卡死、无法翻页 | 待验收 |
| 本地问答 | AI 能基于导入资料回答 | 用户问资料相关问题时，回答来自本地 chunk | 回答泛泛而谈，像普通聊天机器人 | 待验收 |
| 证据来源 | 回答带来源 | 至少显示资料名、页码、原文片段 | 只有自然语言总结，没有出处 | 待验收 |
| 点回原文 | 来源能跳 PDF 页 | 点击来源后打开对应 PDF 页，至少打开 `page_start` | 点击无反应，或只能回到 PDF 首页 | 待验收 |
| 无依据拒答 | 找不到证据时不编造 | 明确说“当前资料库没有找到足够依据” | 编出不存在的论文、页码或结论 | 待验收 |
| 错误提示 | 失败可读 | PDF 解析失败、模型失败、网络失败有明确提示 | 空白、转圈、控制台报错但用户不知道 | 待验收 |

## 3. 试用对象筛选

优先找正在做以下任务的人：

- 开题前快速了解方向。
- 组会或 PPT 汇报。
- 课程论文或期末论文。
- 写 introduction、文献综述或研究背景。
- 围绕“中国道路史”、党史、官方史料、领导人讲话、学术论文等资料找依据。

不要找这些人做第一轮：

- 没有真实论文/资料任务的人。
- 只想看 AI 自动写完整论文的人。
- 只想体验炫酷文献图谱的人。
- 不愿意导入真实 PDF，只愿意看演示的人。

每个用户试用前必须确认：

- 最近 2 周内有真实写作、汇报或资料整理任务。
- 手上有 5-30 篇可合法使用的 PDF 或资料。
- 曾经用过 ChatGPT/Kimi/豆包、浏览器搜索或单篇 PDF 上传工具。
- 曾经因为“AI 没有出处”而需要手动核对。

## 4. 30 分钟试用主持脚本

主持原则：少解释、多观察；不要替用户找按钮；不要问“你喜不喜欢”。

### 0-3 分钟：确认真实场景

主持人说：

```text
今天不是让你评价一个概念，而是请你用自己的真实资料完成一个小任务。你可以像平时一样操作，我主要观察哪里顺、哪里卡。
```

必须问：

1. 你最近一次查资料/写论文/做汇报是什么任务？
2. 你当时用了哪些工具？
3. 最耗时的是找资料、找出处、摘录、引用，还是写作？
4. 你有没有因为 AI 没有出处而不敢用它的回答？

记录重点：用户是否真的有痛点，而不是礼貌配合。

### 3-8 分钟：导入 PDF

给用户任务：

```text
请把你这次任务里真实会用的 PDF 导入系统，数量不用太多，先导入 5-10 篇。
```

观察：

- 用户是否能找到导入入口。
- 是否理解导入成功和解析状态。
- 是否会主动问“导入后的资料在哪里”。
- 是否愿意导入真实资料，而不是随便找测试文件。

不要做：

- 不要替用户提前导入演示资料。
- 不要解释太多系统内部 chunk、向量库原理。

### 8-12 分钟：打开资料并确认原文

给用户任务：

```text
请找到刚导入的一篇资料，打开 PDF，看一下是不是你刚才上传的文件。
```

观察：

- 用户从导入完成到打开 PDF 花了多久。
- PDF 是否能正常阅读。
- 用户是否觉得资料入口自然。
- 用户是否开始把它当成自己的资料库，而不是一次性上传工具。

### 12-22 分钟：围绕真实任务提问

给用户任务：

```text
现在请你围绕刚才的论文/资料，问 3 个你真实会问的问题。尽量问你写论文、做 PPT 或开题时真的需要的问题。
```

可提示的问题类型，但不要直接代写：

- 找出处：这个结论出自哪篇资料？
- 查证据：哪些资料能支持这个观点？
- 做汇报：给我 3 条可以放进 PPT 的观点，每条带出处。
- 课程论文：这个主题有哪些可引用材料？

观察：

- 用户是否自然提出第 2、第 3 个问题。
- 用户是否追问依据。
- 用户是否点击来源。
- 用户是否复制回答或原文片段。
- 用户是否回到原工具核对。

### 22-27 分钟：点击来源核对原文

给用户任务：

```text
请从 AI 回答里挑一个你可能会用进论文或 PPT 的结论，点击来源，回到原文确认它是否真的支持这个结论。
```

观察：

- 来源是否显示资料名、页码、原文片段。
- 点击是否跳到 PDF 对应页。
- 用户是否能在 30 秒内完成核对。
- 核对后用户是否更信任回答。

关键记录：如果跳页失败，本次试用基本不能算通过。

### 27-30 分钟：结束访谈

只问行为问题：

1. 刚才有没有一个回答是你愿意复制到笔记、论文或 PPT 里的？
2. 哪一条来源让你觉得可信？哪一条不可信？
3. 如果明天继续做这个任务，你会不会再打开这个工具？为什么？
4. 你还会继续用 ChatGPT/Kimi/豆包吗？用它们做什么，用 Paper Agent 做什么？
5. 如果这个工具只保留一个功能，你希望是什么？

不要问：

- “你觉得这个产品好不好？”
- “你愿不愿意付费？”除非用户已经出现强使用信号。
- “你想不想要文献图谱/多人协作？”第一轮不要诱导扩功能。

## 6. 用户反馈评分表

每个维度让用户打 1-5 分，但最终判断以行为为主。

| 维度 | 1 分 | 3 分 | 5 分 | 用户评分 |
|---|---|---|---|---|
| 导入清晰度 | 找不到入口 | 能完成但有困惑 | 自然完成并理解状态 |  |
| 资料可找回 | 找不到刚导入资料 | 需要搜索或提示 | 一眼看到最近导入 |  |
| 回答相关性 | 像泛泛聊天 | 部分相关 | 明显基于资料回答 |  |
| 来源可信度 | 没有来源或不敢信 | 有来源但不够清楚 | 来源、页码、片段能支撑结论 |  |
| PDF 回跳体验 | 无法回跳 | 能打开但定位不准 | 直接到对应页并可核对 |  |
| 节省时间感 | 没有节省 | 有一点帮助 | 明显减少找出处/摘录时间 |  |
| 继续使用意愿 | 不会再用 | 特定任务可能用 | 明天愿意继续用同一批资料 |  |

建议再问一个强制排序问题：

```text
如果只能保留一个功能，你选哪个？
A. AI 回答带页码和原文出处
B. 自动总结论文
C. 文献图谱
D. 自动写完整论文
E. 精美笔记系统
```

第一轮期望答案集中在 A。如果用户普遍不选 A，说明当前 MVP 判断可能错了。

## 7. 失败信号

出现以下信号，要暂停加功能，回头修闭环或重做访谈。

### 产品失败信号

- 用户找不到导入入口。
- 用户导入后找不到资料。
- PDF 打不开、空白或无法翻页。
- AI 回答没有来源，或来源不显示页码。
- 来源页码无法点击回 PDF。
- 点击后无法定位，用户仍要手动翻页很久。
- 检索不到时 AI 编造出处。

### 需求失败信号

- 用户不愿意导入真实 PDF。
- 用户只问一个泛泛问题就结束。
- 用户不点击来源，也不关心原文。
- 用户觉得 ChatGPT/Kimi/豆包已经足够。
- 用户想要的是自动写完整论文，而不是查证据。
- 用户第二天没有回来，也没有继续处理同一批资料。

### 定位失败信号

- 用户把它理解成 Zotero 替代品，并主要要求参考文献管理。
- 用户主要要求文献图谱、协作空间、笔记美化。
- 用户不在意页码和原文片段。
- 用户更关心生成一篇完整文章，而不是查找可信依据。

## 8. 下一轮迭代决策规则

第一轮建议测试 5 人。每个人都必须带真实资料和真实任务。

### 继续当前路线

满足以下条件，继续推进“证据型资料问答”：

- 5 人中至少 3 人愿意导入真实资料。
- 5 人中至少 3 人完成 3 个真实问题。
- 5 人中至少 3 人点击来源回看原文。
- 至少 2 人复制回答或原文片段到笔记、PPT 或论文。
- 至少 2 人在 24 小时内继续使用或主动反馈。

下一步迭代重点：

1. 提高证据卡片质量。
2. 提高 PDF 跳页准确性。
3. 增加权威资料类型标识。
4. 增加批量导入和资料过滤。

### 缩小 ICP

如果只有 1-2 人通过，但他们集中在同一场景，例如课程论文或党史资料整理，则不要扩功能，而是缩小目标用户。

可选 ICP：

- 课程/期末论文用户。
- 开题前资料整理用户。
- 组会/PPT 汇报用户。
- 文史/思政主题证据查找用户。

下一步迭代重点：只为这个 ICP 做更短路径。

### 回炉重做

如果 5 人中 0 人通过，或者用户普遍不关心出处和页码，则停止当前 MVP 路线。

回炉动作：

1. 重新访谈用户最近一次查资料过程。
2. 找出他们真正愿意改变工作流的节点。
3. 暂停开发文献图谱、协作、自动写作。
4. 重新定义产品一句话定位。

## 9. 第一轮明确不测的功能

第一轮试用不要测试，也不要主动展示：

- 复杂文献图谱。
- 多人协作。
- 自动写完整论文。
- 精美笔记系统。
- 大规模自动爬虫。
- 复杂标签体系。
- 引文格式自动排版。

如果用户主动提到这些功能，只记录原话，不承诺开发。第一轮唯一要验证的是：

> 带页码和原文片段的证据型回答，是否足以让用户把 Paper Agent 放进自己的资料工作流。

## 10. 试用结论填写模板

每轮 5 人试用结束后填写。

| 指标 | 目标 | 实际 | 判断 |
|---|---:|---:|---|
| 愿意导入真实资料人数 | >= 3/5 |  | 通过 / 不通过 |
| 完成 3 个真实问题人数 | >= 3/5 |  | 通过 / 不通过 |
| 点击来源人数 | >= 3/5 |  | 通过 / 不通过 |
| 成功跳回 PDF 页人数 | >= 3/5 |  | 通过 / 不通过 |
| 复制回答/片段人数 | >= 2/5 |  | 通过 / 不通过 |
| 24 小时内继续使用人数 | >= 2/5 |  | 通过 / 不通过 |

最终结论三选一：

- 继续当前路线：证据型资料问答成立。
- 缩小 ICP：只服务最强场景。
- 回炉重做：当前假设不成立。
